マッチ設定
Wait ?キューに入ってから次の試合が開始するまでの待ち時間（分）。
ポイント変動後この時間は Lobby ステートと推定します。
目安：約5分 m
Match ?1ラウンド（R1）の平均所要時間（分）。
Lobby 終了からこの時間内は In Match (R1) と推定します。
※1試合 = R1+R2+Final の全体。このMatch値はR1基準。
目安：約31分 m
+jitter ?試合時間の誤差マージン（分）。
Match+jitter を超えると In Match (R2) 推定に移行します。
試合が長引いた際のバッファとして機能します。 m
Advanced 設定
リーダーボード ?Embark 公式リーダーボードAPIの取得設定です。
Season ?取得するリーダーボードのシーズン。
S9 = 現行シーズン。シーズンが変わったら更新してください。
Platform ?プラットフォーム別のリーダーボードを選択。
crossplay：全プラットフォーム統合（推奨）
steam / psn / xbox：各プラットフォーム専用
Interval ?APIポーリング間隔（秒）。
短いほどリアルタイムですが API 負荷が増えます。
推奨：60秒
Reflect X ?ゲーム内でポイント変動してからAPIに反映されるまでの遅延時間（分）。
ポイント変動後この時間は POST_MATCH_WAIT 状態と推定。
Update Estimator が自動検出して更新します。目安：8分 m
Tournament ?1大会全体の想定時間（分）。
全ラウンドの合計時間。この時間を超えると Final Round → Offline に移行します。
目安：45分（Wait+Match×1〜2R） m
RS変動アラート ?ポーリングごとにRSの急落を検出して通知します。BANや大量RS喪失の早期発見に使えます。
📉 急落しきい値 ?1回のポーリングでこの値以上RSが減少した場合にトースト通知。
例：1000 → 1000RS以上の減少で通知
BANを受けると数千RS喪失することが多い。 RS以上の減少で即通知
Update Estimator ?Embarkのリーダーボードはバッチ処理で更新されます（約48分周期）。
このツールはそのタイミングを自動検出し、Reflect X を自動更新します。
3回以上バッチを観測すると精度が上がります。
バッチ更新間隔を自動検出・Reflect X を自動更新
Window start ?バッチ検出に使うリーダーボードの開始順位。
例：2000 → 2001位以降を監視。
上位すぎると試合中のポイント変動と区別しにくくなります。
Window size ?バッチ検出に使う件数。
例：500 → 500件を監視。多いほどバッチ検出が安定します。
Cache ?リーダーボードAPIのキャッシュ有効期間（秒）。
Interval より短い値にすると毎回新鮮なデータを取得できます。s
